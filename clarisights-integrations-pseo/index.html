What The Team Needs to Provide

To build these pages at the quality described above, the following inputs are needed from the team.

  1. Full integration list with categories. Every source and destination Clarisights supports, tagged by category (Social Advertising, Search, Attribution, etc.). This becomes the master list for all ~300 source pages.
  2. Top 20 source prioritization. Confirm which 20 integrations are P0 based on customer usage, sales pipeline data, and strategic importance.
  3. Report types per source. For each P0 source, which report types do customers actually build? CS team and demo playbooks are the primary source here.
  4. Customer source combinations. Which sources do real customers use together? This feeds the Cross-Channel Pairings component. HubSpot deal data + CS knowledge.
  5. Metrics and dimensions per source. What data does Clarisights pull from each source? API-level detail for P0 sources.
  6. Industry-source mapping. Which industries use which sources most? Needed for the Industry Use Cases tabs.
  7. Competitive intelligence access. Continued access to Zulip, HubSpot deal data, CS demo playbooks, and customer feedback. This is what makes every component intelligence-informed rather than generic.
  8. Destination capabilities. For each destination, what export options exist? Schema, format, refresh frequency, setup requirements.
  9. Brand assets. Integration logos, Clarisights brand guidelines for page design, approved customer logos for social proof.
  10. Comparison page approval. Sign-off on competitive claims for vs pages (Funnel, Supermetrics, Datorama, Improvado). Legal review if needed.
