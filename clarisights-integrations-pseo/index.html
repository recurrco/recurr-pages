Three Page Types

Each page type serves a different intent. Sources capture the biggest keyword volume. Destinations target dashboard seekers. Connectors catch the long tail.

Type A · Data Source Integration Pages /data-source/{slug}/ · ~300 pages

Every integration Clarisights supports gets a page. Not a "we connect to X" spec sheet — a report-first page that shows what you can build with this data source. 10 components per page, each sourced from customer intelligence.

Report Gallery Cross-Channel Pairings Industry Use Cases Competitive Displacement Intel-Informed FAQ

Destinations Grid — where to export this data
Related Sources — real customer pairings, not "same category"
Intelligence-Informed FAQ — real sales questions, not generic
CTA Bar — demo request + displacement CTA
Why this is different from Funnel
Funnel's integration pages show supported fields and a connector spec sheet. Our pages show what reports you can build, which industries use this source, and why Clarisights beats the native dashboard and competitors. Every component is sourced from customer intelligence — Zulip, HubSpot, CS demos.
Type B · Destination Dashboard & Export Pages /data-destination/{slug}/ · 15-25 pages

Every export target gets a page: Looker Studio, BigQuery, Snowflake, Google Sheets, Power BI, Tableau, and more. The core value prop: "15 connectors vs 1 feed." One unified, normalized data connection replaces a mess of individual connectors.

Dashboard Gallery 15 vs 1 Comparison Source Grid Report Templates

Targets the massive "marketing dashboard examples" cluster at 135K volume/month. Each page shows real dashboard templates and sample report data for that specific destination tool.

Type C · Connector Source-to-Destination Pages /connect/{source}-to-{destination}/ · 50-100 pages

Specific A→B combination pages. "Facebook Ads to Looker Studio," "Google Ads to BigQuery," "TikTok Ads to Snowflake." Each page includes a specific report template for this exact combo. Selective — only validated combinations make the cut.

What The Team Needs to Provide

For Recurr to build these pages at the quality described above, the Clarisights team needs to supply the following.

  1. Full integration list with categories. Every source and destination Clarisights supports, tagged by category (Social Advertising, Search, Attribution, etc.). This becomes the master list for all ~300 source pages.
  2. Top 20 source prioritization. Confirm which 20 integrations are P0 based on customer usage, sales pipeline data, and strategic importance.
  3. Report types per source. For each P0 source, which report types do customers actually build? CS team and demo playbooks are the primary source here.
  4. Customer source combinations. Which sources do real customers use together? This feeds the Cross-Channel Pairings component. HubSpot deal data + CS knowledge.
  5. Metrics and dimensions per source. What data does Clarisights pull from each source? API-level detail for P0 sources.
  6. Industry-source mapping. Which industries use which sources most? Needed for the Industry Use Cases tabs.
  7. Competitive intelligence access. Continued access to Zulip, HubSpot deal data, CS demo playbooks, and customer feedback. This is what makes every component intelligence-informed rather than generic.
  8. Destination capabilities. For each destination, what export options exist? Schema, format, refresh frequency, setup requirements.
  9. Brand assets. Integration logos, Clarisights brand guidelines for page design, approved customer logos for social proof.
  10. Comparison page approval. Sign-off on competitive claims for vs pages (Funnel, Supermetrics, Datorama, Improvado). Legal review if needed.
