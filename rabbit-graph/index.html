// The Problem

The internet is a spatial field of buyer destinations. B2B content marketing operates blind because no tool models it as a connected system.

Existing SEO tools give dashboard reports. Ahrefs shows backlinks. Semrush shows keyword gaps. But none build a persistent, versioned, multi-signal graph of how URLs relate to each other across hyperlinks, search results, AI citations, and brand mentions.

Rabbit maps the terrain first. Then turns it into work.

Edge Type Details

🔗

Hyperlinks

URL A links to URL B. Weighted by source DA, link position, anchor text quality, topic match. The commodity edge — table stakes.

Source: DataForSEO Backlinks API

Phase 1
🔍

SERP Co-occurrence

A and B rank for the same search queries. TF-IDF bipartite projection from SERP data. Rabbit's novel edge type — nobody else builds this.

Source: DataForSEO SERP/Labs API

Phase 1
📖

Co-citation

A and B are both linked from the same third-party page. The internet discusses them together — even if they never link to each other.

Source: DataForSEO Backlinks Intersection

Phase 2
🤖

LLM Citation

A and B cited in the same AI response (ChatGPT, Perplexity, Gemini). 48% of AI citations come from outside organic top 10.

Source: Sellm API (6 platforms)

Phase 3
💬

Entity Co-occurrence

A and B's brands mentioned in the same articles, posts, reviews. PMI-normalized, sentiment-aware.

Source: DataForSEO + Exa + Firehose

Phase 4

Composite Edge Weight

  R(url_A, url_B) = 0.40 x SERP_co-occurrence     // strongest competitive signal
                  + 0.25 x co_citation              // established web relationships
                  + 0.20 x LLM_citation             // emerging AI signal
                  + 0.15 x entity_co-occurrence      // broad brand perception

  Weights configurable per client in Config > Settings.

// SERP Co-occurrence: The Novel Edge

Two URLs that rank for the same search query are in a competitive relationship — even if they never link to each other. This is Rabbit's most unique insight.

  HOW IT WORKS: Bipartite Projection

  STEP 1: Collect SERP data          STEP 2: Build bipartite graph
  (DataForSEO)                       (URLs x Queries)

  Query: "funnel.io alternatives"     Query A ---- url_1 (pos 1)
    #1  clarisights.com/blog          Query A ---- url_2 (pos 3)
    #3  g2.com/compare/...            Query A ---- url_3 (pos 7)
    #7  funnel.io/vs-clarisights      Query B ---- url_2 (pos 2)
                                      Query B ---- url_4 (pos 4)
  Query: "marketing analytics tools"
    #2  g2.com/compare/...            STEP 3: Project to URL-URL graph
    #4  capterra.com/marketing/...
    #9  clarisights.com/blog          url_1 ===== url_2   (strong: shared
                                      url_1 ----- url_3    niche query)
                                      url_2 ----- url_4   (weaker: shared
                                                            broad query)

  FORMULA:
  w(A, B) = SUM over shared queries q:
    rank_score(A,q) x rank_score(B,q) / log(1 + URLs_ranking_for_q)

  where rank_score(url, q) = 1 / log(1 + rank_position)

  WHY THIS WORKS:
  - Niche queries (3 competitors) >> broad queries (50 competitors)
  - Page 1 co-occurrence >> page 3 co-occurrence
  - Newman collaboration weighting prevents dominant query bias

// Authority & Scoring

No single signal determines node importance. Three independent layers, each correcting the others' blind spots.

Google's 2024 API leak confirmed: pagerank_ns (seed distance), siteAuthority (composite), and NavBoost (user behavior) all coexist. Rabbit mirrors this with a blended model.

35%
Graph Authority
Seed distance + betweenness centrality + edge density + DA
35%
Engagement
GSC clicks + position + GA sessions + bounce rate + referral traffic
30%
Content Quality
Depth + freshness + citation density + entity coverage + GEO citability
↓↓↓
node_importance = blended score
↓ 
  PRIORITY FORMULA (what Sujith works on):

  priority = node_importance
           x ring_multiplier        // Core=3, Category=2, Universe=1
           x activation_gap         // how far from desired status
           x (1 + cascade_bonus)    // betweenness: unlocks paths?
           x effort_discount        // Low=1.0, Medium=0.7, High=0.4
           x recency_boost          // recently changed = temp boost

  WHAT "SHIP VERSION" DOES:

  1. Freeze graph snapshot (all nodes, edges, scores)
  2. Rebuild SERP co-occurrence edges (bipartite projection)
  3. Run algorithms (seed-distance, betweenness, Louvain)
  4. Compute priority scores
  5. Generate version diff (vs. previous)
  6. Pull performance data (GA/GSC/HubSpot)
  7. Send automated weekly report
  8. Refresh backlog
  9. Stream counter resets

// Temporal Model

Every edge type decays at a different rate. Hyperlinks are permanent. LLM citations are volatile.

Edge TypeHalf-LifeRationale
Hyperlinks~4 yearsLinks rarely disappear
Co-citation~230 daysEditorial links are sticky
Entity co-occurrence~100 daysBrand perception evolves
SERP co-occurrence~70 daysRankings shift quarterly
LLM citations~14 daysLLM outputs change with retraining
  weight(t) = base_weight x e^(-lambda x days_since_verified)

  Co-citation:       lambda = 0.003    ||||||||||||||||||||||..............
