Mission

One coherent work plane the team can trust

Recurr runs on three surfaces. Plane is the truthful, client-facing view of work. Slack is where everyone talks. Claude is the always-fresh, intelligent workspace where the team actually does the thinking — connected to every secondary tool and every scrap of project context. The Recurrops Agent's whole job is to keep these three in sync and in shape so the team — and eventually a hands-off founder — can rely on them without manual upkeep.

Job Description

Six standing responsibilities

JD-01

Keep Claude fresh

Continuously sync each project's real state (from Plane + Slack) into the Claude-accessible knowledge so the team always opens an up-to-date workspace. Freshness is the core metric.

JD-02

Guard the three-tool plane

Make sure work lives where it should — reality in Plane, talk in Slack, active work in Claude — and that the three never drift apart.

JD-03

Wire secondary tools into Claude

Ensure Sanity, Figma, Google Workspace, GitHub, Metabase and the rest are connected to Claude brilliantly via MCPs / APIs / connectors — so any member can reach any tool from the workspace.

JD-04

Broadcast updates globally

Roll up progress across all projects and broadcast a clean, regular update so nothing happens in a silo.

JD-05

Chase missing updates

When an owner hasn't logged progress or a decision never made it to the board, follow up until it's captured.

JD-06

Enforce planning & cycle compliance

Make sure work is planned properly and cycles/sprints are actually run — flag projects that fall out of rhythm.

Lifecycle

The loop it runs, per project

STEP 1
Observe

Read the Plane board + recent Slack for the project.

STEP 2
Reconcile

Compare reality against what the workspace currently says.

STEP 3
Refresh Claude

Update the project repo so the workspace is current.

STEP 4
Flag hygiene

Surface stale tasks, undefined work, broken cycles.

STEP 5
Chase

Nudge owners for missing updates & decisions.

STEP 6
Broadcast

Post the rolled-up update to the team.

↻ runs on a schedule (daily per project) · global broadcast on its own cadence · state persists between runs

Rollout

Earns trust before it acts

Phase 0
Report-only pilot Clarisights

One project. Reads Plane + Slack, writes only to the repo, posts a summary. Changes nothing in Plane or Slack. We watch it.

Phase 1
Freshness + hygiene, all projects

Daily loop across every project. Keeps Claude fresh and flags hygiene problems. Still read-only on Plane & Slack.

Phase 2
Active broadcast + follow-ups

Now allowed to post the global broadcast and nudge owners in Slack for missing updates.

Phase 3
Compliance + integration automation

Enforces planning & cycle compliance, and automatically wires/repairs secondary-tool connections to Claude.

Guardrails

What it will not do (yet)

  • Never messages clients — internal team only.
  • Plane stays the single source of truth for priorities; the agent reflects it, never overrides it.
  • Starts report-only — no writes to Plane or Slack until each phase is trusted.
  • Destructive or outward actions require a human approval step until proven.
