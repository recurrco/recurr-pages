What this is

A direct line between your AI assistant and your full Clarisights setup.

The Marketing Data MCP is a remote Model Context Protocol server hosted by Clarisights. Once your team adds it to Claude, ChatGPT, Gemini, or any MCP-compatible assistant, that assistant can browse the same data you can — your channels, your dashboards, your custom metrics, your saved segments — and answer questions about live performance in plain language.

There's no CSV export step. No "let me copy this into the prompt." No re-explaining your CAC formula every conversation. The AI uses your team's definitions, scoped to your team's permissions, across every channel Clarisights is wired into — [CONFIRM channel list: Facebook/Meta, Google Ads, TikTok Ads, Snapchat Ads, Pinterest, LinkedIn Ads, Reddit Ads, Microsoft Advertising, + analytics + others].

Authentication is OAuth 2.0. The first time your assistant calls a Clarisights tool, you sign in with your Clarisights account in a browser window. There's nothing to manage — no API tokens, no developer accounts, no rotating credentials. Disconnecting from your AI tool's settings revokes access immediately.

When this is the right tool

Use the Marketing Data MCP when you need an analyst. Use a native MCP when you need an operator.

There's a fast-growing ecosystem of native MCP servers — open-source connectors that wrap the Meta Marketing API or the Google Ads API and hand the keys directly to an AI assistant. They're useful for what they do. They're not what we are. Here's the decision frame.

If you want to… Reach for
Ask "how is spend pacing this week vs. last week, blended across all channels?" → Marketing Data MCP
Have the AI launch a new Meta campaign with a draft creative → Native Meta Ads MCP (e.g. Pipeboard, Meta's official MCP)
Compare CPM across Facebook, TikTok, and Snapchat in one prompt → Marketing Data MCP
Pull a raw GAQL query result from Google Ads for engineering debugging → Native Google Ads MCP
Get answers using your team's custom CAC, blended ROAS, or NTR definitions → Marketing Data MCP
Let the AI mutate budgets, pause campaigns, or change bids → Native MCP (and a careful approval flow)
Share an answer with a colleague who has different permissions → Marketing Data MCP (permissions are inherited)
Build a one-off internal tool against a single ad platform → Native MCP

The simple rule: if the question is about understanding performance across your full marketing stack, this is the right surface. If the question is about executing a single-channel change, a native MCP plus a human approval flow makes more sense. Most paid-media teams need the first kind of answer ten times more often than the second.

Setup — by tool

Under a minute per tool. OAuth, no API keys.

Pick the assistant you already use. Each flow opens Clarisights in a browser window, you sign in, you approve access. That's it.

Claude (web, desktop, mobile)
~45s

Pro, Max, Team, or Enterprise plan.

  1. Open Claude → Settings → Connectors.
  2. Click Add custom connector.
  3. Name: Clarisights. URL:
    https://app.clarisights.com/mcp
  4. Click Add, sign in to Clarisights.
  5. In any chat: + → Connectors → enable Clarisights.
ChatGPT
~60s

Business, Enterprise, Edu, or Pro plan with Developer mode.

  1. Settings → Connectors → Advanced settings → enable Developer mode.
  2. Settings → Connectors → Create.
  3. Name: Clarisights. URL: https://app.clarisights.com/mcp. Auth: OAuth.
  4. Save, sign in to Clarisights.
  5. In a new chat: tools menu → toggle Clarisights on.
Gemini Enterprise
~45s

Workspace / Enterprise plans only.

  1. Settings → Apps & connectors → Custom connectors.
  2. Click Add connector.
  3. URL: https://app.clarisights.com/mcp. Auth method: OAuth 2.0.
  4. Save, sign in to Clarisights.

Consumer Gemini doesn't support custom connectors today.

Cursor
~30s

Cursor Settings → MCP & Integrations → Add new MCP server. Paste the JSON below into the editor and save.

{ "mcpServers": { "clarisights": { "url": "https://app.clarisights.com/mcp" } } }
Claude Code
~10s

One command:

claude mcp add --transport http \ clarisights https://app.clarisights.com/mcp

Sign-in is triggered on first Clarisights tool call.

OpenAI Codex CLI
~20s

Add to ~/.codex/config.toml:

[mcp_servers.clarisights] url = "https://app.clarisights.com/mcp"

Restart Codex; first tool call triggers OAuth.

Any other MCP client that supports remote HTTP servers will work — point it at https://app.clarisights.com/mcp and use OAuth 2.0. For older stdio-only clients, you can bridge with npx mcp-remote.

Why this beats a raw API MCP

The semantic layer is the product. The MCP is just how it gets to your assistant.

A native Meta Ads MCP gives Claude a steering wheel attached to a raw v25 API. The Marketing Data MCP gives Claude a steering wheel attached to your business — your metric definitions, your account groupings, your permissions, your saved views.

What you inherit with a native API MCP

  • One OAuth flow per platform. Six ad accounts means six setups. Meta + Google + TikTok means three different auth dances.
  • Token rotation is your problem. Meta tokens expire in ~60 days. System-user tokens require Business Manager setup. Refresh logic ends up in your prompts.
  • Rate-limit collisions. One LLM prompt can fan out into 30 sub-queries. Meta's per-account, per-BUC, rolling 1h rate limits aren't designed for that.
  • Schema instability tax. Meta removed 100+ metrics from Ads Insights in late 2024. Attribution windows changed in Jan 2026. Google sunsets API versions on a 3-month cycle. Your prompts break with the API.
  • Re-derive every metric, every conversation. What does your team mean by CAC? Media-only? Plus agency? Plus creative? Claude doesn't know — unless you tell it, every time.
  • No cross-channel anything. A Meta-only MCP can't compare Meta to TikTok. A Google-only MCP can't see your blended ROAS.
  • No permission inheritance. The MCP either has full account access or none. There's no "this person only sees the EU accounts."

What you get with the Marketing Data MCP

  • One identity, all channels. Your Clarisights account is the only credential. Every connected platform is reachable in one prompt.
  • No token plumbing. Clarisights handles platform-side auth, refresh, and version migration. You never see a Meta access token.
  • Your team's metric definitions. The CAC, ROAS, MER, and custom metrics your team has built in Clarisights are the metrics the LLM uses. Same numbers as the dashboards.
  • Stable schema across platform changes. When Meta sunsets a field or Google ships v18, Clarisights absorbs it. Your prompts keep working.
  • Cross-channel by default. Blended views, attribution comparisons, cross-platform pacing — first-class concepts, not workarounds.
  • Permissions inherited from the UI. The assistant only sees what the signed-in user can see. No accidental leakage of accounts outside scope.
  • Dashboards & saved segments are addressable. "Open the Q1 review dashboard" works. Years of saved internal IP becomes prompt-accessible.
  • One rate-limit budget. 1,000 queries per user per day, governed centrally, not per-platform.
The position

The analyst, not the operator.

There's a vocal debate inside paid-media teams right now about how much an AI assistant should be allowed to actually do. Should it pause campaigns when CPA spikes? Should it shift budget between ad sets? Should it write new copy and ship it?

The cautious answer is the right one. A bot flipping bids every 15 minutes isn't a superpower — it's a way to confuse the algorithm you're already paying to optimize. And the cost of an AI making the wrong write-back decision is asymmetric: a bad query wastes a chat turn; a bad bid change wastes a week of budget.

The Marketing Data MCP does not write back to ad platforms. By design. This isn't a limitation we'd patch later — it's the position. We think the highest-leverage use of an AI assistant in a paid-media team is to give every person on the team an on-call senior analyst. Not a co-pilot you have to babysit. An analyst who answers when you ask.

If and when you want execution, you reach for a different tool — a native MCP with a real approval workflow, an automation platform like Smartly or Pencil, or a human. The Marketing Data MCP makes sure the question that triggered that decision was answered with the right data, on the right metric definition, in the right amount of time.
